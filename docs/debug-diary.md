# Debug diary

A running log of real bugs hit in the projects in this portfolio,
what they looked like, how they were tracked down, and what fixed
them. Written for future-me, not for readers, but kept public because
an honest record of dead ends is more useful than a polished
success story.

Entries are chronological. Each one is short by design — title, one
paragraph on symptom, one on root cause, one on fix. If an entry is
worth more than that, it belongs in the project's own README.

## Format

```
### <title>

**Project:** tracks/X-.../...
**Symptom:** what broke, how it presented.
**Root cause:** what was actually wrong.
**Fix:** one sentence.
**Time to diagnose:** rough estimate.
**Lesson:** what to check first next time.
```
